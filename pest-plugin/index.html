Pest plugin

Make debt growth fail in CI.

The plugin scans deterministic PHP debt signals, compares them to explicit budgets, and can ratchet against a committed baseline.

Install

composer require --dev odinns/volven-pest-plugin

Basic Pest Test

it('keeps debt inside the budget', function (): void {
    debt()->budget()
        ->phpstanIgnores(max: 12)
        ->todoComments(max: 20)
        ->mixedTypes(max: 12)
        ->largeClasses(max: 5)
        ->mustNotGrow()
        ->requireReasons()
        ->assert();
});

Supported Signals

  • PHPStan suppression annotations: `@phpstan-ignore`, `@phpstan-ignore-line`, `@phpstan-ignore-next-line`, and `@phpstan-ignore-*`.
  • TODO and FIXME comments.
  • `mixed` types.
  • Large class-like PHP files.

Baseline And Ratchet

Generate `.volven/debt-baseline.json` from PHP, commit it, then use `mustNotGrow()` in the budget test. Current MVP baseline generation is intentionally plain PHP, not a rich CLI.

debt()->budget()
    ->phpstanIgnores(max: 999)
    ->todoComments(max: 999)
    ->mixedTypes(max: 999)
    ->largeClasses(max: 999)
    ->writeBaseline();

Reason Checks

`requireReasons()` fails PHPStan suppressions with fewer than three reason words. It does not apply to TODOs, `mixed`, or large classes.

Tool Boundaries

Vølven is not Pint, PHPStan, Psalm, Rector, Pest `arch()`, or a codemod. It tracks debt pressure. PHPStan type-shaping annotations such as `@phpstan-type`, `@phpstan-import-type`, `@phpstan-template`, `@phpstan-var`, `@phpstan-param`, and `@phpstan-return` are ignored because they describe contracts instead of suppressing them.

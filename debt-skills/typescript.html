TypeScript skill

Static types are not runtime contracts.

Use this skill where TypeScript is green but external data, casts, duplicated state, or UI logic still make the next change risky.

Signals

  • `any`, `unknown as`, non-null assertions, and casts that hide uncertainty.
  • API, local storage, URL, and event payloads trusted without validation.
  • State split across component, store, cache, URL, and server without one owner.
  • React components owning business rules that should live behind a boundary.
  • Mocks that prove implementation details while the real integration can break.

Boundary

The skill does not replace ESLint, TypeScript, Prettier, codemods, framework checks, or architecture rules. It reviews what remains after those are done.

Best Use

Use it on API edges, React state changes, auth and billing flows, cache logic, schema migrations, and build config changes.

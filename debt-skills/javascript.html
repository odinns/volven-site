JavaScript skill

Runtime shape is the contract.

Use this skill when the expensive failures will come from missing contracts, hidden mutation, async drift, or framework behavior nobody has written down.

Signals

  • Objects whose expected shape only exists in caller memory.
  • Callback and promise mixing that hides ordering or error behavior.
  • Mutable shared state touched from unrelated paths.
  • Divergent helper functions that encode competing business rules.
  • Framework magic without local conventions or tests around the edge.

Boundary

The skill does not replace ESLint, Prettier, framework checks, or codemods. It looks for runtime debt those tools will politely walk past.

Best Use

Use it on browser integrations, Node services, shared utilities, async workflows, legacy modules, and framework-heavy code.

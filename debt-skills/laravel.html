Laravel skill

Review the framework magic before it becomes folklore.

Laravel makes useful things easy. It also makes some side effects very quiet. This skill looks in those quiet corners.

Signals

  • Model observers, events, and casts hiding writes or external calls.
  • Queued jobs without idempotency, retry shape, or failure ownership.
  • Validation, policies, resources, and database constraints drifting apart.
  • Migrations that are hostile to production data, locks, or rollbacks.
  • Factories and feature tests that know more rules than the app code.

Boundary

The skill does not replace Pest architecture tests, PHPStan, Psalm, Rector, Pint, or Laravel's own checks. It reviews the future cost those tools cannot price cleanly.

Best Use

Use it on PRs that touch models, jobs, migrations, authorization, validation, billing, mail, queues, scheduled tasks, or framework upgrades.
